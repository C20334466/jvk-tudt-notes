2 Create and deploy a small application of your own using Angular

Your application can have any functionality but must use at least one instance of each Angular feature covered in the tutorial. The tutorial is structured to cover one feature per page, which will help you tick off the lab requirements.

To create a new Angular project, use:

ng new project_name

To run the application for testing, use:

ng serve --open --poll=2000

Deploy your Angular application to GitHub pages. You will find help on this here. This article covers the whole process from scratch, which means that you will only need parts of it. You will need to adjust the directory names for the structure of your RWAT repository (do not create a sub-repository inside the RWAT repository!).

 

IMPORTANT NOTES

  • Make sure to add a file called .gitignore with the line 
    
**/node_modules/*
    This will prevent the node_modules directory from being unnecessarily uploaded to GitHub.

    If the file already exists in your RWAT repository, then add the line to the end of the file without deleting existing content.

  • Do not include the tutorial project in your RWAT repository. Only include the project directory of your own application.
  • No marks will be awarded if the two instructions above are not complied with.
