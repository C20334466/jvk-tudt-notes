School of Computer Science

Forms

Assignment

  1. Create a HTML file and in it create a form (see documentation) for user registration at a discussion board website.
  2. The form should consist of three sections, implemented with the use of fieldset elements.
  3. All fieldset elements should contain a legend (see this article section about the two element types on MDN).
  4. All fields in all sections should have corresponding labels (see this article section on labels on MDN).
  5. The first section includes personal information fields:
    username (required, should be validated not to be empty)
    A text input
    password (required, should be validated to be strong enough)
    A password input, which should be 8 to 12 characters long and contain at least one of: a lower case letter, an upper case letter, a digit and a special character out of !@#%&*_=+-$^ (you can use the pattern attribute of the text input element, with the value (?=.*[a-z])(?=.*[A-Z])(?=.*[0-9])(?=.*[!@#%&*_=\+\-\$\^]).{8,12})
    email address (required, should have its format validated)
    An input element of type email
    date of birth (should have its format validated)
    A date input in the format DD/MM/YY and an explanation as to why the date of birth is needed and how it will be used (see this article under Why Do You Need This Information?)
  6. The second section includes information on boards the user is interested in:
    purpose
    A set of checkboxes indicating the purpose of joining the website (work, hobbies, entertainment)
    boards of interest (required, should be validated to include at least one choice)
    A multi-select drop-down list of board topics, which allows the selection of more than one topic (at least one must be selected)
  7. The third section includes volunteering information:
    volunteering availability (should be set to 'not available' by default)
    A set of radio buttons whereby the user can select a level of voluntary work on the website they are prepared to offer ('none', '1 hour', '2-3 hours', 4-5 hours)
  8. Information on how to indicate that a field is required can be found here. Fields will be validated according to their type or given pattern (as for password).
  9. Style the form with CSS (not Bootstrap or other framework) and make it user friendly.
  10. Place the form inside a dialog element, which is open when the HTML page is loaded (see dialog documentation for more information on the dialog element and its open attribute).
  11. The form should have a submit button that can be clicked to cause a submit event and the closing of the dialog (see form documentation for information on the form element and its method attribute, as well as button documentation for information on the button element and its type attribute).
  12. Write a JavaScript function that handles the submit event by gathering all the form data and issuing a confirmation of registration, including a listing of all the included data.
