  2. Introduction - read carefully!

    Lab work contributes 5% to the final result in CSWD. To receive marks for a lab in any week of the term, the submitted work must be:

    • uploaded before the beginning of the following week's lab
    • be in a folder (not zip file) named labN (where N is the lab number, e.g. lab1 or lab2)
    • contain evidence of a reasonable effort to complete the lab

  3. How to upload lab work?

    All lab work is to be uploaded to your private repository in GitHub Classroom "CSWD - Yr 2 Computing TUDT 2019/20". To create this repository, do the following:

    1. In order to be able to access this GitHub classroom, you must have a GitHub account. If you don't have an account with GitHub, the first thing you should do is to create an account at github.com.
    2. Once you have an account with GitHub, you can create a repository in the GitHub classroom and prepare it for use. Complete the following steps to achieve this:
      1. Open the following URL in a browser: https://classroom.github.com/a/Pgjo7M2U
      2. Log in if you are prompted (if you are already logged into GitHub in the browser, you will not be prompted)
      3. Click on the green button Accept this assignment when prompted
      4. The creation of your classroom repository will proceed automatically, at the end of which you should see this message:
      5. Click on the bottom link (after the text "Your assignment has been created here:") to access your new repository
      6. Now, to initialise the repository you should add a README file to it by clicking README:
      7. The README file is opened and it contains some default text. If your GitHub account name does not identfy you, type your name into the README file to let the tutor know who you are. Then click on Commit new file to commit the README file to the repository
      8. Copy or bookmark the link to this repository, as this is where you will be uploading all your lab work for Client-Side Web Development
