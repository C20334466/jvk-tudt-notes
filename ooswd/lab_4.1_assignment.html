Notation in assignment specifications
  • Angular brackets containing descriptive text are placeholders, used to describe required input or other literals. For example,
    • if an expected input string value is described as <name> then the actual value could be John or Mary
    • if a value is described as <date of birth in format dd/mm/yy> then the actual value might be 01/01/01

    The actual value should not include angular brackets.

  • If a command line window is shown to demonstrate requirements,
    • text printed by the programe is displayed in white
    • user input is displayed in red
    For example,
    Please enter an number: 42

General instructions

  • Include comments in your Python scripts. These don't need to be overly verbose, but sufficient for a reader to understand the thinking behind the code.
  • Input/output
    • Unless otherwise specified, you can assume a well behaved user, who always enters valid input.
    • Reading of user input must always be accompanied by a suitable textual prompt.
    • Printing of output must always be accompanied by a suitable textual heading or explanation.

Exercise 1: Random experiment

  1. Write some code in Python that performs the following experiment:
    • random numbers in the range [1, 10] (1 to 10 inclusive) are drawn until 10 is drawn
    • the result is the number of draws it took to get a 10
  2. Use a suitable Python loop to perform this experiment 100 times, recording the results in a list.
  3. Print the average result i.e. the mean of the number of times it took to draw a 10.
  4. Ask the user if they would like the data printed out and, if the answer is positive, print the data using a function.
