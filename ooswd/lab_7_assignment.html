Lab assignment 7: Exception handling

Assignment

  1. Starting with your solution (or the given one) to exercise 3 in lab assignment Revision, strings, lists (week 2), modify it so that:
    1. user input errors are handled, either with conditional statements or exceptions, as deemed appropriate
    2. the DRY (don't repeat yourself) principle is applied (e.g. the code for processing own and friend's preferences should not be repeated and the printing of items should use a loop)
    3. the number of items should be configurable with the following constant, defined in the first line of the script file:

      REQ_LIST_LEN = 5

      and the program should work for any value of the constant
  2. Starting with your solution (or the given one) to lab assignment Classes (week 5), modify it so that the following errors are handled with the use of exceptions:
    1. the product name passed into the Product constructor is an empty string
    2. the stock value passed into the Product constructor is not a non-negative integer
    3. the product name passed into the remove_product is not on the product list

    Write some code in main.py to test the error handling.
