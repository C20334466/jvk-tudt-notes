EXERCISE 1

Define three classes called Source, Upper and Storer.

The classes have the following methods:

  • Source.do_work() produces a random number (between 1 and 10) of random letters ('a' to 'z') and passes them (one by one) to an instance of Upper by calling Upper.queue_work()
  • Upper.queue_work() adds the received letters to a deque object inside the Upper object
  • Upper.do_work() converts a random number (between 1 and 10, clipped to the queue length) of queued letters to uppercase, taking them one-by-one according to the first-in-first-out (FIFO) pattern from the deque object and passing them converted to the Printer object by calling Printer.queue_work()
  • Storer.queue_work() adds the received letters to a deque object inside the Storer object
  • Storer.do_work() adds a random number (between 1 and 10, clipped to the queue length) of the queued letters into a store of type set, taking them one-by-one accoring to the first-in-first-out (FIFO) pattern from the deque object
  • Upper.queue_length() and Storer.queue_length() each return the length of their inner deque object
  • Storer.get_store() returns a list of the stored values, alphabetically ordered

The orchestration code should run a for loop e.g. 100 times, calling do_work() on the Source, Upper and Storer objects (in that order) in each iteration, and printing the length of the queues in every iteration.

At the very end, it should print the letters of the alphabet not stored in the Storer object.

EXERCISE 2

Define a class called Dictionary that stores words and their definitions internally in an object of type dict.

Use the following data as the initial content of the dictionary

{
"apple": "a fruit, can be green, red or yellow",
"baby": "a very young person", 
"car":	"a transport vehicle",
"deer":	"an animal with antlers",
"earth": "a planet, the one we live on"
}

The Dictionary class should:

  • allow the look-up of a word
  • allow the addition of words to the dictionary, one or more at a time
  • by default, issue a warning, without updating the word, if the addition of an existing word is attempted
  • provide a mechanism for the above default behaviour to be overridden, so that a word can be updated if needed
  • allow the deletion of a word, issuing a suitable message if the word is not in the dictionary

Define a class called DictionaryUI, which handles user interactions with an object of the Dictionary class, allowing them to make full use of the functionality of that class. It should have a function called run which kicks-off the interaction.

In the top-level scope there should be code that creates and connects a Dictionary and a DictionaryUI object and calls run on the DictionaryUI object.

Complete the entire exercise in a single Python script file.
