Week 4 Lab 2: Online shop

Your assignment is to write two Python programs for online shopping at a company called "Groceries at your Fingertips". Part 1 describes the requirements for the first program. Part 2 describes requirements for the second, which is an extended version of the first program.

PART 1

In this part you will be developing a system that allows a user to place a grocery order.

  • Assumptions

    • The system may assume that all user input will be correct.

  • Requirement: Menu

    • The system should have a Shop menu with 5 items representing types of groceries of your choice (e.g. bread, milk) and additional items called submit order and exit
    • The menu should have a suitable header and include instructions to users.
    • Each menu item should be displayed alongside a choice key string that the user can key in to choose that item (e.g. 1, 2, 3 etc.)

  • Requirement: Actions

    Actions should be implemented to handle menu items as follows:
    MenuItemAction
    Shop[any grocery item]

    Display information about the grocery including a short description and price.

    Ask the user to enter a quantity (integer) for the grocery, save the information and re-display the Shop menu. Make sure to handle the quantity of 0.

    Shopsubmit order

    Calculate the price of the order.

    Display the list of bought items in descending order of price per item.

    Display a suitable message about the total price and delivery (we assume that the user is logged in and their address is known).

    Exit the program.

    Shopexit

    Exit the program.

  • The program should start by displaying the Shop menu.

  • Hints

    • Use lists to store grocery items and prices. You might use three one-dimensional lists or one two-dimensional list.
    • Hard-code the 5 grocery items and prices.

  • Sample interaction

PART 2

In this part you will be extending the system so that it keeps track of stock.

Start by copying your Python file from Part 1. Use the copy to complete Part 2.

  • Requirement: Additional feature - stock tracking

    Modify the program to keep track of available quantities of the groceries. The new feature should include:

    • storing the quantity in stock for each grocery item (at the beginning this should be set to 10 for each item)
    • informing the user of the quantity available when they are asked to enter the quantity they wish to buy
    • decreasing the quantity by the number specified by the user, while handling the cases where the user enters invalid numbers (negative or greater than the number in stock) in some reasonable way
    • displaying out-of-stock items in the Shop menu with
      • an out-of-stock marker beside the item
      • without the choice key string
