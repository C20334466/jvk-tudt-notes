Week 4 Lab 3: Stock management

Your assignment is to write a Python programs for keeping track of items sold in a shop called "Groceries at your Fingertips", fullfilling the following functional requirements:

  • User interaction

    • Except where indicated, your program may assume that user input is going to be valid (i.e. error handling should be implemented only if stated as an explicit requirement).
    • Where user input is a functional requirement, suitable prompts must be provided, e.g. Please enter your name:
    • Where output is a functional requirement, suitable descriptions must be provided, e.g. Your order total is €100 rather than just €100.
    • Communication needs to be maintained with the user for a good user experience, which means that output should not be limited to that specified in the functional requirements. For example, if an item is deleted as requested by the user and the program remains 'silent', the user will be left wondering what happened. A message such as Item successfully deleted would contribute to a better user experience.

  • User interface

    The user interface is a page consisting of two parts:

    1. A list of the items stocked by the shop (at least two items should be pre-loaded at program start-up e.g. bread and milk), each with the following information:
      • unique code (a positive integer)
      • name of item
      • price
      • number in stock
    2. A menu listing the following options available to the user (a shop employee):
      • add item
      • remove item
      • update item
      • update stock
      • exit
      Each menu option should be displayed with a code to be used for choosing the option (e.g. 1, 2, 3, 4).

  • Actions

    • On startup, the program should display the user interface page.
    • Actions should be implemented to handle the four menu options as follows:
      Menu option Action
      add item

      Read in user input consisting of information about the item, including name, price and number in stock.

      Assign a unique id to the item.

      Redisplay the user interface page with the list updated to include the new item.

      remove item

      Read in user input consisting of an item's unique id.

      Remove the item associated with the id from the list of items.

      Redisplay the user interface page with the list updated not to include the item specified for deletion.

      update item

      Read in user input consisting of an item's unique id.

      Read in user input consisting of information about the item, including name, price and number in stock.

      Redisplay the user interface page with the list updated to reflect the new information about the item specified for update.

      update stock

      For each item get a random number between 0 and the number in stock and reduce the number in stock by that random number.

      Display a list of items for which stock is low (2 or less), ask user to press any key to continue and read this input.

      Redisplay the user interface page with the list updated to reflect new stock information.

      exit

      Exit the program.
