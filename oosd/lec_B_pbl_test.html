A Brief High-Level Look at Programming in Python

Operations considerations

  • Project structure (modules, makefile, test material etc.)
  • Source control (how the structure is going to be managed)
  • Collaboration (if more than one person involved)
  • Integration
  • Testing
  • Deployment

Design considerations

  • Analysis and interpretation of requirements
  • High-level design
    • required data structures (database? what type?)
    • program flow
      • run-through
      • event-handling
  • Several iterations of the two if the problem is complex
  • This applies in agile as much as it did in the 'waterfal' development methodology (but on a smaller scale)
  • Early test identification at any level
  • Low-level design
  • Coding

Problem-based learning

  • PBL starts with the process, rather than details of how to do particular tasks
  • Top-down approach, but on a much smaller scale than for a 'real' project.
